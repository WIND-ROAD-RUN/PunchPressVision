# 创建模型（CreateModel）功能需求文档

> 本文档汇总 PunchPressVision 中"创建模型"界面的全部功能需求与实现细节。
> 最后更新：2026-06-17

---

## 1. 运行模式

### 1.1 CreateModel 运行模式

- **枚举值**：`global::RunMode::CreateModel`（定义于 `global/include/global/GlobalType.hpp`）
- **相机行为**：关闭触发模式（Free Run），持续取流，图像实时刷新到 ShapeEditor 控件
- **切换逻辑**：`app::PunchPressApp::configureCameraForCreateModel()` 复用 Debug 模式的自由运行配置，`switchToMode()` 处理相机启停
- **进入/退出**：`ModelEditorDialog` 构造时切换到 `CreateModel`，析构时切回 `Idle`

### 1.2 相机图像跨线程传输

- **问题**：相机线程 → 主线程的 `QueuedConnection` 因缺少类型注册导致图像无法到达 UI
- **修复**：`UI/src/main.cpp` 中注册 Halcon 类型：
  ```cpp
  qRegisterMetaType<HalconCpp::HImage>("HalconCpp::HImage");
  qRegisterMetaType<global::CameraIndex>("global::CameraIndex");
  ```
- **信号链**：`TwoCameraSpliceInfTool`（相机线程）→ `CameraBun` → `PunchPressApp::frameReady` → `ModelEditorDialog::onFrameReady` → `ShapeEditor::displayImage`

## 2. ShapeEditor 控件（L3 复合控件）

### 2.1 架构

- **三层结构**：
  - L1 `HalconDisplayLabel`：Halcon 原生窗口容器
  - L2 `HalconInteractiveLabel`：缩放/平移交互
  - L3 `ShapeEditor`：ROI/Mask/标记叠加层 + 工具切换
- **渲染引擎**：Halcon 原生绘制（`DispObj`, `DispRectangle1`, `DispCross`, `OverpaintRegion`, `WriteString`）

### 2.2 工具类型

| 工具 | 枚举值 | 说明 |
|------|--------|------|
| 视图 | `Tool::View` | 默认模式，支持缩放/平移 |
| 感兴趣区域 | `Tool::RectangleROI` | 绿色矩形绘制 |
| 屏蔽区域 | `Tool::RectangleMask` | 品红色矩形绘制 |
| 中心点 | `Tool::CenterPoint` | 蓝色十字标记 |

### 2.3 颜色方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| ROI 已确认 | `green` | 已完成的感兴趣区域矩形 |
| ROI 拖拽中 | `yellow` | RectangleROI 工具橡皮筋 |
| Mask 已确认 | `magenta` | 已完成的屏蔽区域矩形 |
| Mask 拖拽中 | `orange` | RectangleMask 工具橡皮筋 |
| 中心点 | `blue` | DispCross 十字标记 |
| 识别标记 | `#00FF00` | 匹配成功后绿色十字 + 分数文本 |

### 2.4 多 ROI / 多 Mask 支持

- **数据结构**：
  - `QVector<QRectF> roiRects_` — 所有 ROI 矩形
  - `QVector<QRectF> maskRects_` — 所有 Mask 矩形
  - `QVector<ActionType> actionHistory_` — 统一回撤栈（LIFO）
- **导出接口**：
  - `roi()` / `mask()` — 将所有矩形合并为单个 `HalconCpp::HObject` 区域
  - `roiRects()` / `maskRects()` — 返回原始矩形列表
- **批量操作**：
  - `clearAll()` — 清空 ROI + Mask + 中心点 + 识别标记
  - `undo()` — 弹出 actionHistory_ 最后一条记录并移除对应数据

### 2.5 绘制取消机制

- **Escape 键**：事件过滤器中捕获 `Qt::Key_Escape` → 取消当前拖拽 + `setTool(Tool::View)`
- **Toggle 按钮**：再次点击绘制按钮回到 View 模式，按钮文字动态切换：
  - 绘制感兴趣区域 ⇄ 退出绘制
  - 绘制屏蔽区域 ⇄ 退出屏蔽
  - 定义中心点 ⇄ 退出定义

### 2.6 识别标记

- **存储**：`showMarker_`, `markerRow_`, `markerCol_`, `markerAngle_`, `markerScore_`
- **绘制**：`drawRecognitionMarker()` — 绿色十字（60px 半径）+ 分数文本
- **清除**：`clearMarker()` — 隐藏标记，下次 `refreshOverlay()` 时不再绘制
- **UI 操作**："识别"按钮成功后绘制；"清空绘制区域"时清除

### 2.7 防止栈溢出

- **问题**：`displayImage → imageLabel_->displayImage → viewChanged → refreshOverlay → displayImage(imageLabel_->lastImage())` 形成递归
- **修复**：
  1. `displayImage()` 中直接调用 `drawAllROIs()` / `drawAllMasks()` / `drawCenterPoint()` / `drawMarker()`，不调用 `refreshOverlay()`
  2. `displaying_` 重入哨兵：`refreshOverlay()` 检查 `displaying_` 标志，防止递归调用

## 4. 数字键盘交互

- **统一使用**：`rw::rqwu::NumberKeyboard::inputDataOnQPushButton()`
- **整数参数**（增益/曝光/核大小/对比度）：
  ```cpp
  rw::rqwu::NumberKeyboard::InputDataConfig cfg;
  cfg.isUsingMin = true;
  cfg.isUsingMax = true;
  cfg.min = min;
  cfg.max = max;
  ```
- **相机参数同步模式**：使用 C++ 成员指针语法 `int Config::cameraCfg::* member` 实现 DRY：
  ```cpp
  void applyExposure(global::CameraIndex idx, int value, int Config::cameraCfg::* member);
  void applyGain(global::CameraIndex idx, int value, int Config::cameraCfg::* member);
  ```
- **拒绝即无操作**：`result != NumberKeyboard::Accept` → 直接返回，不修改原值

## 5. 识别功能（创建前预览匹配）

### 5.1 流程

1. 校验 `requireROI("识别")` — 无 ROI 则弹窗提示并中止
2. 校验 `lastFrame_.IsInitialized()` — 无图像则提示"请先载入图像"
3. 通过 `buildRequest()` 构造 `CreateModelRequest`
4. 调用 `ShapeModeManagerBun::testRecognize()` — 临时创建模型 → `FindShapeModel` → 销毁临时模型
5. 匹配成功：`ShapeEditor::drawRecognitionMarker()` 绘制绿色十字 + 分数
6. 匹配失败：`ShapeEditor::clearMarker()` + 弹窗提示

### 5.2 按钮位置

- `btn_recognize` 定义在 `Dlg_createshapemodel.ui` 中 `verticalLayout_4` 最上方
- 在"创建模板"按钮之上

## 6. 创建模型（持久化）

### 6.1 前置校验

- `requireROI("创建模型")` — 无 ROI 则弹窗提示并中止
- 屏蔽区域缺失不影响创建（仅 ROI 为必需）

### 6.2 CreateModelRequest 参数

```cpp
struct CreateModelRequest {
    HalconCpp::HImage trainingImage;     // 训练图像
    HalconCpp::HObject roi;              // ROI 合并区域
    HalconCpp::HObject mask;             // Mask 合并区域
    QVector<QRectF> roiRects;            // ROI 矩形列表（逐个保存支持回撤）
    QVector<QRectF> maskRects;           // Mask 矩形列表
    QPointF centerPoint;                 // 手动中心点
    bool hasCenterPoint;
    QString name;                        // 模型名称（空则用时间戳）
    double exposure, gain;               // 创建时相机参数
    bool upperLight, lowerLight;         // 光源状态
    int imageChannelType;                // 图像通道类型
    bool useOpening;  int openingSize;   // 开运算
    bool useClosing;  int closingSize;   // 闭运算
    bool useMean;     int meanSize;      // 均值滤波
    int contrast;                        // 对比度（0=自动）
    int minContrast;                     // 最小对比度
    // ... 其他 Halcon 训练参数
};
```

### 6.3 数据持久化到 ShapeModelData

创建模型时，`ShapeModeManagerBun::createModel()` 将以下参数写入 `ShapeModelData`：

| 字段 | 来源 |
|------|------|
| `_paintCreateRoiList` | 逐个 ROI 矩形 → `HalconCpp::HObject` 列表 |
| `_paintShieldRoiList` | 逐个 Mask 矩形 → `HalconCpp::HObject` 列表 |
| `_SingleChannelType` | `comboBox_ImageType.currentIndex()` |
| `_createModelPreProcessType` | 同上 |
| `_createModelUseOpening` | `ckb_opening.isChecked()` |
| `_createModelOpeningRadius` | `openingSize_` |
| `_createModelUseClosing` | `ckb_closing.isChecked()` |
| `_createModelClosingRadius` | `closingSize_` |
| `_createModelUseMean` | `ckb_mean.isChecked()` |
| `_createModelMeanRadius` | `meanSize_` |
| `_createModelExposureTime` | 相机曝光值 |
| `_createModelGain` | 相机增益值 |
| `maxContrast` / `minContrast` | 对比度参数 |
| `_templateMatImage` | 模板训练图像 |
| `_originalImage` | 原始图像 |
| `_annotatedImage` | 带 ROI/Mask 标注的预览图 |

## 7. 图片存储

### 7.1 文件命名与格式

| 文件 | 格式 | 说明 |
|------|------|------|
| `template_image.jpg` | JPEG (quality 90) | 模板训练图像 |
| `original_image.jpg` | JPEG (quality 90) | 原始图像 |
| `annotated.jpg` | JPEG (quality 90) | 标注图像（ROI=白色边框, Mask=黑色填充） |
| `model.shm` | Halcon 格式 | 形状模型 |
| `metrology.mmc` | Halcon 格式 | 计量模型 |
| `paint_create_roi_N.hobj` | Halcon 格式 | 第 N 个 ROI 区域 |
| `paint_shield_roi_N.hobj` | Halcon 格式 | 第 N 个 Mask 区域 |
| `model_params.txt` | 键值对 | 所有参数 |
| `model_info.json` | JSON | 模型元信息 |

### 7.2 格式切换

- **从 BMP 切换到 JPEG**：
  ```cpp
  HalconCpp::SetSystem("jpeg_quality", 90);
  HalconCpp::HImage(image).WriteImage("jpeg", 0, filePath);
  ```
- Halcon `WriteImage` 签名：`WriteImage(Format, FillColor, FileName)` — 3 个参数，非 4 个
- JPEG 质量通过 `SetSystem("jpeg_quality", value)` 设置

### 7.3 标注图像生成

- 使用 Halcon `OverpaintRegion`：
  - ROI 区域 → 白色边框（RGB 255,255,255）叠加
  - Mask 区域 → 黑色填充（RGB 0,0,0）叠加
- 仅当用户绘制了对应区域时才生成标注

### 7.4 存储路径

- **路径格式**：`{RootPath}/ShapeModels/{YYYYMMDDHHmmssSSS}/`
- **修复前**（Bug）：`{RootPath}/ShapeModels{YYYYMMDDHHmmssSSS}` — 缺少 `/` 分隔符
- **修复**：`RootPath + "/" + currentTime` ← 在拼接处插入 `/`

## 8. ROI 校验

### 8.1 requireROI()

```cpp
bool ModelEditorDialog::requireROI(const QString& action) const {
    if (shapeEditor_ && shapeEditor_->hasROI())
        return true;
    rw::rqwu::MessageBox::warning(this, "提示",
        QStringLiteral("%1前请先绘制感兴趣区域（ROI）").arg(action));
    return false;
}
```

- **调用位置**：
  - `onRecognize()` — "识别前请先绘制感兴趣区域（ROI）"
  - `onCreateModel()` — "创建模型前请先绘制感兴趣区域（ROI）"
- **不要求 Mask**：仅 ROI 为必需条件，屏蔽区域可省略
